Built by the `appr_kitchen_month` CTE in `sp_rebuild_me_bridge.sql`. Feeds `live_sold_rate_approved` (Live - Sold Rate with Approved %).

**Definition evolution (Jun-Jul 2026), 3 steps:** (1) static snapshot bug -> (2) monthly FLOW (approved IN that month; tied to the Approved-Deals report) -> (3) cumulative still-approved-TODAY (Jad Jul 1: "carry over MoM, but only ones still in Approved") -> (4) FINAL (Jad Jul 2: "accurate for current month but not right" + Maysam "keep working on A, keep cumulative"): **Approved WINDOW gate** = `date_approved <= month_end AND exit_d > month_end` where `exit_d = CASE WHEN stage_now='approved' THEN 9999-12-31 WHEN closed_won_date__c NOT NULL THEN it WHEN closed_lost_date__c NOT NULL THEN it ELSE COALESCE(laststagechangedate, date_approved) END`. History now keeps deals that were pending THEN even if closed since (step-3 gate dropped them retroactively: ME Jan 2 vs true 8).
